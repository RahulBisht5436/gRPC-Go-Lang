# `proto/greet.pb.go` — Generated message structs

> ⚠️ Iss file ko **kabhi haath se mat edit karo**. File ki pehli line khud kehti hai:
> ```go
> // Code generated by protoc-gen-go. DO NOT EDIT.
> ```
> `make generate` chalaane se ye dobara generate hoti hai. Manual changes lost ho jaayenge.

## Ye file kya hai?

`protoc-gen-go` plugin ne `greet.proto` ke saare **`message`** declarations ko Go code me convert kiya. Yaha kuch business logic nahi hai — bas data structures (structs) aur unke helpers.

`service`-related code alag file me jaata hai: [`greet_grpc.pb.go`](./greet_grpc.pb.go.md) — vo `protoc-gen-go-grpc` plugin banata hai.

## Struct fields ko samjho

```go
type GreetRequest struct {
    state         protoimpl.MessageState   // protobuf internal
    FirstName     string                   // <-- tumhara field
    unknownFields protoimpl.UnknownFields  // protobuf internal
    sizeCache     protoimpl.SizeCache      // protobuf internal
}
```

- **`FirstName`** — yahi tumhare proto wala field hai. Capital `F` se start hota hai (Go me **exported** hone ke liye PascalCase chahiye).
- Baaki 3 fields (`state`, `unknownFields`, `sizeCache`) **protobuf runtime ke internals** hain. Inko ignore karo. Ye fields backward/forward compatibility, lazy parsing, aur reflection ke liye use hote hain.

### Struct tag dekho

```go
FirstName string `protobuf:"bytes,1,opt,name=firstName,proto3" json:"firstName,omitempty"`
```

Ye **struct tag** hai (backticks me). Iska decode:

| Part | Matlab |
|---|---|
| `protobuf:"..."` | Protobuf encoder/decoder ke liye instructions |
| `bytes` | Wire type (length-prefixed bytes — strings ke liye) |
| `1` | Field number (proto se: `string firstName = 1`) |
| `opt` | Optional (proto3 default) |
| `name=firstName` | Original proto name (camelCase) |
| `proto3` | Proto syntax version |
| `json:"firstName,omitempty"` | JSON serialization ke liye (gRPC-Web ya REST-via-grpc-gateway) |

## Helper methods generate hue

Har message ke saath ye methods automatic banti hain:

### 1. `Reset()` — message ko zero value pe le aata hai

```31:36:gRPC project/greet/proto/greet.pb.go
func (x *GreetRequest) Reset() {
	*x = GreetRequest{}
	mi := &file_greet_proto_msgTypes[0]
	ms := protoimpl.X.MessageStateOf(protoimpl.Pointer(x))
	ms.StoreMessageInfo(mi)
}
```

Pool-based memory reuse ke liye useful. Tum directly call nahi karoge mostly.

### 2. `String()` — debug format

```38:40:gRPC project/greet/proto/greet.pb.go
func (x *GreetRequest) String() string {
	return protoimpl.X.MessageStringOf(x)
}
```

`fmt.Println(req)` ya `log.Printf("%v", req)` jab use karte ho, isi ka output dikhta hai. Output dikhta hai `first_name:"Rahul"` jaisa.

### 3. `ProtoMessage()` — marker method

```42:42:gRPC project/greet/proto/greet.pb.go
func (*GreetRequest) ProtoMessage() {}
```

Bas ek **empty marker** method. Ye kahin call nahi hota — sirf type system ko bata raha hai "ye proto message hai". Interface satisfaction ke liye.

### 4. `ProtoReflect()` — reflection API

```44:54:gRPC project/greet/proto/greet.pb.go
func (x *GreetRequest) ProtoReflect() protoreflect.Message {
	mi := &file_greet_proto_msgTypes[0]
	if x != nil {
		ms := protoimpl.X.MessageStateOf(protoimpl.Pointer(x))
		if ms.LoadMessageInfo() == nil {
			ms.StoreMessageInfo(mi)
		}
		return ms
	}
	return mi.MessageOf(x)
}
```

Advanced — protobuf reflection ke liye. Tools jaise `grpcurl`, debug proxies, schema-based code is method ka use karte hain. Tumhare normal handler code me iska direct istemaal nahi.

### 5. `GetXxx()` — nil-safe getters (sabse important!)

```61:66:gRPC project/greet/proto/greet.pb.go
func (x *GreetRequest) GetFirstName() string {
	if x != nil {
		return x.FirstName
	}
	return ""
}
```

Yeh **important** hai. Tumhare handler me always `in.GetFirstName()` use karo, **direct `in.FirstName` nahi**. Reason:

```go
// In nil-able pointer
var req *pb.GreetRequest = nil

req.FirstName       // ❌ panic: nil pointer dereference
req.GetFirstName()  // ✅ returns "" safely
```

Ye specially nested messages me bachata hai. Imagine:

```proto
message Order {
    Customer customer = 1;
}
message Customer {
    string name = 2;
}
```

Generated:

```go
order.GetCustomer().GetName()  // safe even if customer is nil
order.Customer.Name            // panics if customer is nil
```

## File ke neeche wala "magic" code

	"\fGreetRequest\x12\x1c\n" +
	"\tfirstName\x18\x01 \x01(\tR\tfirstName\"'\n" +
	"\rGreetResponse\x12\x16\n" +
	"\x06result\x18\x01 \x01(\tR\x06result2B\n" +

Ye **raw protobuf descriptor** hai — tumhare proto file ka encoded form. Reflection, dynamic decoding, aur `grpcurl` jaise tools is descriptor ka istemaal karte hain.

## Beginner ke liye yaad rakhne wali baatein

| Sawaal | Jawab |
|---|---|
| Ye file mai edit kar sakta hu? | **Nahi.** `make generate` regenerate karega aur changes mit jaayenge. |
| `firstName` proto me, `FirstName` Go me — kyu? | Go me exported field PascalCase me hote hain. `protoc` automatic convert karta hai. |
| `GetFirstName()` ya `FirstName` — kya use karu? | Hamesha `GetFirstName()` — nil-safe hai. |
| `protoimpl.MessageState`, `unknownFields`, `sizeCache` kya hai? | Protobuf runtime internals. Ignore karo. |
| Ye sab generate kab hota hai? | Jab tum `protoc --go_out=. greet.proto` chalao (ya `make generate`). |

## Proto change to flow kya hai?

```
1. Edit greet.proto (e.g., add `string last_name = 2;` to GreetRequest)
2. cd gRPC project/greet
3. make generate    <-- ye greet.pb.go ko regenerate karega
4. Ab Go code me `req.GetLastName()` available hoga
5. Server/client code me usage update karo
```

Bilkul iterative loop hai — proto edit, regenerate, code update.
